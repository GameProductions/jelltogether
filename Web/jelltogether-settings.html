Discord Stage

Configure one server-wide Discord bot connection for watch party stage topic sync.

Requirements
You must create your own Discord bot for JellTogether to use this feature. The bot must be invited to the Discord server that contains the Stage channel. The bot token must belong to that bot and be saved here before channels can be discovered. The bot needs permission to view channels, read message history, send messages, and manage the selected Stage channel topic. Only Discord Stage channels are eligible for sync. Treat the bot token like a password. Tokens entered here are stored in Jellyfin plugin configuration. For stronger storage, set JELLTOGETHER_DISCORD_BOT_TOKEN on the Jellyfin server or container instead of saving a token here.
Create And Connect A Discord Bot
Open the Discord Developer Portal and create a new application. Open the application's Bot page, add a bot, then copy or reset the bot token and paste it into Bot Token below. Open OAuth2 URL Generator, select the bot scope, then grant View Channels, Read Message History, Send Messages, and Manage Channels permissions. Use the generated invite URL to add the bot to the server that owns your Stage channel. In Discord, make sure the bot role can view the Stage channel, read and send messages there, and manage that channel. For secure server-side storage, set the JELLTOGETHER_DISCORD_BOT_TOKEN environment variable and restart Jellyfin. Save settings, then use the Stage Channel picker to select an eligible Stage channel.
